Definition of Done – Checklist Técnico
1. Código e Qualidade

 Todo o código foi revisado (Code Review feito e aprovado).

 O código segue o padrão de estilo definido (lint, formatação e convenções).

 Não há warnings ou erros de compilação.

 Há tratamento de exceções e logs adequados.

 Não há código morto ou comentários desatualizados.

 O repositório está organizado e limpo, sem arquivos temporários.

4. Build e Deploy

 O projeto compila sem erros.

 O build está automatizado via CI/CD.

 O deploy está automatizado ou com instruções testadas.

 O ambiente de produção está configurado e validado.

 O rollback ou recuperação de erro está documentado.

5. Funcionalidades

 Todos os requisitos funcionais definidos foram implementados.

 Todos os critérios de aceite foram verificados pelo Product Owner.

 O sistema entrega valor perceptível ao usuário final.

 Nenhum bug conhecido bloqueia o uso das funções principais.

6. Segurança e Desempenho

 Dados sensíveis estão criptografados e protegidos.

 Acesso e autenticação funcionam corretamente.

 Não há credenciais expostas no repositório.

 Foram feitos testes de desempenho mínimos (tempo de resposta, carga).

 O sistema lida bem com erros e falhas de rede.

7. Acessibilidade e Usabilidade

 Interface segue critérios de acessibilidade básicos (contraste, navegação, texto alternativo).

 Fluxos de uso foram testados por alguém fora da equipe.

 Feedback do usuário foi coletado e documentado.
